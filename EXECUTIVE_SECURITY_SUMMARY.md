## ⚠️ **核心發現：3 個致命漏洞 + 4 個高危漏洞**

### 🔴 **致命漏洞（立即修復）**

#### 1. **API 密鑰硬編碼洩露** 💥
- **風險:** Gemini API key 暴露在源代碼中
- **影響:** 攻擊者可以無限調用 API，造成費用激增
- **修復:** 環境變數化（30 分鐘）
- **優先級:** 🔴 P0 - **本周完成**

#### 2. **JWT 密鑰硬編碼洩露** 💥
- **風險:** 任何人都可以偽造任意用戶 token
- **影響:** 完整繞過身份驗證，冒充任何用戶（包括管理員）
- **修復:** 環境變數化（30 分鐘）
- **優先級:** 🔴 P0 - **本周完成**

#### 3. **Token 存儲於明文 localStorage (XSS 漏洞)** 💥
- **風險:** 任何網站的 XSS 攻擊都可竊取用戶 token
- **影響:** 帳號被冒充長達 30 天（JWT 過期時間）
- **修復:** 改用 HttpOnly Cookie（2 小時）
- **優先級:** 🔴 P0 - **本周完成**

## 📊 **用戶隱私威脅等級**

### 🚨 **最嚴重的隱私威脅**

#### 1️⃣ **健康敏感數據完全暴露**
```
威脅: 所有用戶的血壓、血糖、用藥記錄、服用時間以明文存儲
影響: 資料庫洩露 → 完整的醫療檔案暴露
違法: GDPR €20M 罰款 + 台灣個資法 NT$5-50M 罰款
```

#### 2️⃣ **Token 被竊取，帳號被冒充**
```
威脅: XSS 攻擊可竊取 localStorage 中的 token
影響: 攻擊者可以冒充用戶讀取/修改所有健康數據
持續時間: 30 天（直到 JWT 過期）
```

#### 3️⃣ **CORS 允許任意網站存取**
```
威脅: evil.com 上的惡意代碼可以竊取用戶數據
影響: 所有登入的用戶都被暴露
攻擊難度: 非常容易（只需一個 fetch 調用）
```

#### 4️⃣ **備份文件明文存儲**
```
威脅: 備份中的所有數據也是明文
影響: 備份洩露 = 完整的歷史健康檔案洩露
```
